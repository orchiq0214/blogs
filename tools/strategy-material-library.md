# 策略素材库：把拍脑袋变成可验证的决策资产

## 一句话

策略素材库不是替人自动制定策略，而是把每次策略灵感、实验设计、结果和复盘沉淀下来，让下一次决策不再从零开始。

它适用于推荐、运营、增长、商业化等持续试错场景。

```text
发现问题
  -> 找到可调动作
  -> 组合成策略
  -> 灰度实验
  -> 推全或回退
  -> 结果回流素材库
```

## 为什么需要它

业务策略常来自经验、临场判断或少数人的记忆。这个过程并不一定错误，但有三个问题：

- 同类问题重复出现时，团队仍然从头想办法。
- 策略为什么有效、对谁有效、代价是什么，很难被复用。
- 实验结果散落在报告、群聊和人的记忆里，无法反哺下一轮决策。

目标不是消灭人的判断，而是让判断有可检索的先例、可配置的执行方式和可追溯的结果。

## 三层资产

### 1. 动作库：有哪些杠杆可以调

动作库记录系统允许人或机器调整的能力，而不是泛泛的建议。

以内容推荐为例，可以包括：

- 流量位置、候选池、排序权重和用户分层。
- 曝光频控、重复推荐冷却期、负反馈后的降权或暂停。
- 推荐文案、专题组织、转化链路和商业化约束。
- 灰度范围、观察指标、异常阈值和回滚动作。

动作必须最终落到配置界面、规则引擎或可调用接口；否则它只是文档，不是决策系统。

### 2. 策略素材库：在什么条件下怎样组合动作

一条策略素材不是“把权重调高”这样的孤立动作，而是一个可复用的决策单元：

```text
触发场景：什么现象、什么人群、什么约束下出现
假设：认为哪个机制导致了问题
策略组合：调用哪些动作，参数如何设置
预期收益：希望改善什么指标，允许付出什么代价
风险与边界：不适用什么场景，谁需要审批
实验设计：对照组、灰度范围、观察周期、成功/失败阈值
历史证据：曾经的实验结果、置信度、失败原因
```

它比案例库多了一层“可执行配置”，比规则库多了一层“问题背景与实验证据”。

### 3. 实验与复盘库：策略到底有没有用

策略上线后，系统需要记录：

- 实际触发条件与配置版本。
- 实验分组、样本量、观察周期和核心指标变化。
- 收益、代价、异常、投诉或负反馈。
- 最终决策：推全、继续实验、调整参数或回退。
- 复盘结论：策略适用边界、失败原因和下次应如何修改。

结果不能只写成“实验成功/失败”。它要回写到对应策略素材，改变下一次推荐时的证据强度。

## 产品闭环

```text
问题信号 / 人工需求
  -> 策略助手检索相似素材与历史证据
  -> 生成策略草案和实验方案
  -> 人审核并在配置台确认
  -> 灰度执行与监控
  -> 达标推全；异常自动或人工回退
  -> 结果、反例和复盘进入素材库
```

这里的 AI 负责加速三个环节：

- 从问题描述中召回相似策略、失败案例和适用边界。
- 根据动作库生成可审核的策略与实验配置草案。
- 汇总实验结果，提示是否达到推全、调整或回退条件。

AI 不应绕过权限直接改线上策略，也不应把相关性叙述成因果结论。

## 最小可行版本

第一版不需要自动执行策略，可以先做一个人工主导的闭环：

1. 建 20-30 条高频动作及其配置边界。
2. 用统一模板沉淀历史策略和实验复盘。
3. 支持按问题、目标指标、人群、动作和实验结果检索。
4. 生成策略草案，但必须由人补全实验阈值并确认。
5. 将最终结果回写为“有效条件 / 无效条件 / 待验证”。

验证标准不是“AI 能生成多少策略”，而是：面对一次新问题，团队能否更快找到合理的候选方案，并且减少重复试错和不可追溯的拍脑袋。

## 长期价值

模型能力会越来越容易获得；难以复制的是组织在真实业务中积累的动作空间、约束、实验历史和责任边界。

策略素材库把这些经验变成一种持续增长的决策资产：既服务人，也让未来的 AI 有证据可循、有边界可守。
